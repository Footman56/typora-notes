http 与https 最大的区别就是安全

| 特性         | HTTP                       | HTTPS                                                        |
| :----------- | :------------------------- | :----------------------------------------------------------- |
| **协议基础** | 运行在 TCP 协议之上        | **运行在 SSL/TLS 协议之上**，再由 SSL/TLS 运行在 TCP 协议之上 |
| **数据传输** | **明文传输**（文本形式）   | **加密传输**（密文形式）                                     |
| **安全性**   | 低。易遭受窃听、劫持、篡改 | 高。提供身份验证、数据加密和数据完整性校验                   |
| **默认端口** | 80                         | **443**                                                      |

HTTPS 并不是一种新的应用层协议，它只是在 HTTP 接口下面加了一层安全套接层（SSL，Secure Sockets Layer）或其继任者传输层安全协议（TLS，Transport Layer Security）。

工作流程：

1. 身份验证：当你访问一个 HTTPS 网站时，服务器会先把自己的**数字证书** 发给你
2. 浏览器检查证书是否有效
3. 加密通信
   1. 非对称加密： 浏览器使用证书中的公钥，加密一个用于后续通信的**对称密钥**发送给服务器
   2. 对称加密：双方使用刚协商好的对称密钥，对即将传输的HTTP数据进行加密和解密



为什么需要https

1. 防止隐私泄露
2. 防止内容劫持和篡改
3. 身份验证
4. SEO 排名和用户信任 ：主流搜索引擎会给予HTTPS 网站更高的搜索排名

# MQ异步更新

1. 写请求到redis 中
2. 发送MQ消息
3. MQ消息写数据库

消费时检查数据是否更新《保证幂等》



单队列+单消费者

1. 确保队列只被一个消费者监听。
2. 在消费者代码中，**严禁使用多线程处理**，在回调函数里老老实实同步处理。
3. 关闭自动 ACK，使用手动 ACK，且处理成功后才 ACK。

多队列+ 业务分区

1. **生产者**：根据业务 ID（如 `order_id`）计算哈希值，决定发送到哪个队列。
   - 例如：`order_queue_0`, `order_queue_1`, `order_queue_2`。
2. **消费者**：每个队列对应一个消费者（或者一个消费者的单线程），保证每个队列内部是顺序的。

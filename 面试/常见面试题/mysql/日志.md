# redo log

- **保证事务的持久性（Durability）**：当事务提交时，即使系统突然崩溃，已提交的事务修改也不会丢失。
- **物理日志**：记录的是“数据页（Page）做了哪些修改”，例如“将表空间XX中的页YY的偏移量ZZ处的值改为AAA”。

- **redo log**：是 **InnoDB 存储引擎层** 特有的日志。

# binlog

记录了所有对数据库执行更改的操作（包括数据更新和结构变更），但不包括`SELECT`和`SHOW`这类查询操作

**binlog**：是 **MySQL Server 层** 实现的日志

# undo log

会滚日志

- **保证事务的原子性（Atomicity）**：当事务需要回滚时，能够恢复到修改前的状态。
- **实现MVCC（多版本并发控制）**：为一致性非锁定读（如普通的`SELECT`）提供历史数据版本，避免脏读、不可重复读。
- **逻辑日志**：记录的是数据的逻辑变化，例如“将某条记录的某个字段从值A改为值B”的反向操作。

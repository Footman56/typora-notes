# 二、大致流程

**尽可能的复现操作流程**

sentry 群

```
1、根据异常的堆栈信息，确定出问题的代码
2、查看代码，确定问题的可能来源
3、根据traceId 来确定问题链路，从链路中获取关于公司，方案的信息
4、查看数据库中方案信息是否正确，数据库中的数据是否符合预期
5、可以根据请求链路的拦截，确定好操作对象。（进可能的复现操作的流程）
```

bug 反馈群

```
在mis 后台研发自助中 根据手机号查看公司信息
根据客户提供的情景，查看客户操作时请求的接口
```

 

**绩效 以方案为核心，关注方案的状态**。

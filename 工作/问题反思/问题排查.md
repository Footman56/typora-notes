





# 三、死锁

1. 首先先定位是哪些sql 导致的死锁，一个巴掌拍不响。「有死锁就说明，是团伙作案」

2. 根据sql 定位代码位置、**事务开始的位置**

3. 死锁的本质是 两个事务相互拿着对方需要的资源。整理每个事务获取的资源，参考如下图：

	<img src="/Users/peilizhi/Library/Application Support/typora-user-images/image-20211223184958755.png" alt="image-20211223184958755" style="zoom:50%;" />

4. 梳理清楚资源关系，之后就需要调整代码执行的顺序，保证事务更新的顺序

5. 调整业务，有顺序的执行业务，避免多个事务之间的竞争

**归根结底都是需要定位业务问题。**

#### 当一个事务太大的时候 对于同时插入 或者 同时更新 等操作要仔细观察 

可能发生死锁的情况：

- 高并发同时操作同一条数据
- 存在主键和辅助索引，加锁顺序相反

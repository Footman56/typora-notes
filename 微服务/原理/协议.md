# TCP/IP 

TCP/IP 是两个协议，IP用于定位服务器地址，对应网际互联层；

TCP 用于规范传输的格式，对应的是传输层

![image-20230129144347441](https://gcore.jsdelivr.net/gh/Footman56/imageBeds/202301291443476.png)

网络接入层：将需要互相连接的节点接入到网络，为网络传输提供基础

网际互联层：找到目标节点，并且传输到目标节点 (IP)

传输层：负责对报文进行分组和重组，并以TCP或UDP协议格式封装报文。 （TCP）

应用层：使用数据(HTTP、SMTP、FTP)

## 三次握手

TCP 协议是**安全的、面向连接的、可靠的字节流**服务

![tcp header](https://gcore.jsdelivr.net/gh/Footman56/imageBeds/202301291546650.jpg)

- 序列号（seq）：用于对报文进行计数（注`SYN`和`FIN`都会消耗一个序列号），`TCP`为应用层提供全双工服务，连接的每一端都要保持每个方向上的传输序列号
- `SYN`：用来发起一个连接，当新建一个链接时，`SYN`变为1
- `ACK`：确认位，ACK=1 ,确认号才起作用。正常通信之后ACK=1,第一次因为没有收到确定ACK=0。
- 确认号（ack）；接受端告诉发送端已经成功接受到的数据，并且期望下次收到的数据，值为上次收到的序列号+1

在传输数据之前通过三次握手来建立全双工的链路。

三次握手流程就是**发送-确认-再发送**

如果服务器确认时候没有再收到数据，就会再次发送确认。

![image-20230129154320482](https://gcore.jsdelivr.net/gh/Footman56/imageBeds/202301291543539.png)

四次挥手就是客户端和服务器各自向对方发送一次请求，请求都被确认之后才结束。

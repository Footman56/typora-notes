# 单台机器搭建集群

想要构成集群的话就需要启动多个elasticsearch 实例，使用一个配置文件，并且在启动的时候单独指定运行端口，数据存储位置，节点名称

1. 在配置文件中去掉单节点的标志信息

   ```yaml
   #http.port: 9200
   #cluster.initial_master_nodes: ["node-1"]
   #node.name: node-1
   ```

2. 启动命令

   ```shell
   ./bin/elasticsearch  -d
   
   ./bin/elasticsearch -Ehttp.port=9201 -Epath.data=/home/huochai/elasticsearch-node2/data -Epath.logs=/home/huochai/elasticsearch-node2/log -Enode.name=node-2 -d
   
   ./bin/elasticsearch -Ehttp.port=9202 -Epath.data=/home/huochai/elasticsearch-node2/data -Epath.logs=/home/huochai/elasticsearch-node2/log -Enode.name=node-2 -d
   ```

   

ES集群架构的优势:

+   提高系统的可用性，部分节点停止服务，整个集群的服务不受影响
+  存储的水平扩容
